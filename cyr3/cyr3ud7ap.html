UD7. Desarrollo Web

UD7. Desarrollo Web y Animación Digital

La World Wide Web ha dejado de ser un simple repositorio de documentos estáticos para convertirse en un ecosistema dinámico y complejo.

Para un estudiante de Computación y Robótica, comprender cómo se construye y se sirve el contenido en la red es fundamental, ya que la web es la interfaz principal a través de la cual interactuamos con sistemas inteligentes, bases de datos y dispositivos remotos.

En esta unidad, analizaremos desde la arquitectura interna de un sitio web hasta las tecnologías que permiten dar vida a las interfaces mediante la animación, finalizando con la implementación práctica en entornos de desarrollo modernos.

Análisis de la estructura de las páginas web

Toda página web que visualizamos en nuestro navegador es el resultado de una arquitectura organizada en capas.

Si comparamos una web con un edificio, el HTML sería la estructura de vigas, el CSS la pintura y decoración, y el JavaScript los sistemas automáticos (como el ascensor o los sensores de luz).
  • 1. Capa de Estructura (HTML): El lenguaje de marcas de hipertexto define qué hay en la página. Utiliza etiquetas para identificar títulos, párrafos, imágenes y enlaces. Sin esta estructura, el navegador no sabría qué información mostrar.
  • 2. Capa de Presentación (CSS): Define cómo se ve la información. Se encarga del diseño, los colores, las tipografías y la disposición de los elementos (layout). Es lo que permite que una web sea visualmente atractiva y adaptable a móviles (Responsive Design).
  • 3. Capa de Comportamiento (JavaScript): Es el lenguaje de programación que añade interactividad. Permite que ocurran cosas sin recargar la página: desplegar menús, validar formularios o mostrar animaciones complejas.

Nota técnica:

La organización jerárquica de una web se conoce como DOM (Document Object Model). Es una estructura en forma de árbol que el navegador genera para representar todos los elementos de la página y poder manipularlos en tiempo real.

Anatomía de una página web: Header, Body y Footer

Más allá de las capas de lenguajes (HTML, CSS, JS), una página se organiza visualmente en bloques funcionales que el usuario identifica de forma instintiva.

1. El Header (Cabecera)

Es la parte superior de la página y lo primero que carga el navegador. Su función es puramente de identificación y navegación. Elementos clave: El logotipo de la empresa o sitio, el menú de navegación principal (links a otras secciones), un buscador y, en ocasiones, botones de llamada a la acción (CTA como "Iniciar sesión" o "Comprar"). Importancia técnica: Suele ser consistente en todas las subpáginas del sitio para que el usuario no se pierda. En diseño moderno, se utiliza a menudo el "Sticky Header", que se queda fijo en la parte superior mientras haces scroll.

2. El Body / Main Content (Cuerpo o contenido principal)

Es el bloque central y el más variable. Aquí es donde reside la información específica que el usuario ha venido a buscar. Estructura interna: Se organiza mediante etiquetas semánticas como <section>, <article> y <aside> (barra lateral). Contenido dinámico: Es la zona donde se insertan las imágenes, vídeos, textos detallados y las animaciones que hemos analizado anteriormente. Su diseño debe priorizar la legibilidad.

3. El Footer (Pie de página)

Se encuentra en el extremo inferior y cierra la estructura del sitio. Aunque suele recibir menos atención, es vital para la credibilidad y el cumplimiento legal. Elementos clave: Información de contacto, enlaces a redes sociales, mapa del sitio web, avisos legales (política de privacidad y cookies) y créditos de copyright. Función técnica: Actúa como una red de seguridad; si el usuario llega al final de la página y no ha encontrado lo que buscaba, el footer le ofrece rutas alternativas.

Tabla Comparativa: Secciones Estructurales

Sección Ubicación Función Principal Elemento Representativo
Header Superior Identificación y acceso rápido. Logotipo y Menú.
Body Central Entrega de contenido y valor. Artículos, imágenes y multimedia.
Footer Inferior Información legal y secundaria. Copyright y redes sociales.

Nota técnica: En el desarrollo con HTML5, estas secciones no son solo visuales. Existen etiquetas específicas (<header>, <footer>, <main>) que ayudan a los motores de búsqueda como Google a entender qué parte de la web es la más importante, mejorando el posicionamiento SEO.

Sabías que...

Existe una zona llamada "Above the fold" (por encima del pliegue). Es todo lo que el usuario ve en el Header y el inicio del Body sin necesidad de hacer scroll. Es la parte más crítica de una web: si no convence en menos de 3 segundos, el usuario suele abandonar el sitio.

Servidores web: tipología

Para que una página web esté disponible 24/7 en cualquier lugar del mundo, debe estar alojada en un servidor web. Un servidor es, esencialmente, un ordenador de altas prestaciones conectado permanentemente a Internet cuya función es "servir" (enviar) los archivos de la web al navegador del usuario (cliente) cuando este los solicita. Existen diferentes tipos de servidores según cómo gestionan sus recursos:

Tipo de Servidor Descripción Ventajas Inconvenientes
Servidor Compartido Varias webs comparten los recursos (CPU, RAM) de una misma máquina. Económico y fácil de mantener. Si una web recibe mucho tráfico, puede ralentizar a las demás.
Servidor VPS (Virtual Private Server) Un servidor físico se divide en varios servidores virtuales independientes. Mayor control y recursos garantizados. Requiere más conocimientos técnicos para su gestión.
Servidor Dedicado Una máquina física completa para un solo cliente o proyecto. Máximo rendimiento y seguridad total. Coste muy elevado.
Cloud Hosting (La Nube) La web se aloja en un conjunto de servidores interconectados. Escalabilidad inmediata y alta disponibilidad. El coste puede variar según el consumo de recursos.
Sabías que... Cuando escribes una dirección web (URL), un sistema llamado DNS (Domain Name System) traduce ese nombre a una dirección IP numérica, que es la dirección real del servidor donde están los archivos.

Formatos de animación web

La animación en la web no solo cumple una función estética, sino que mejora la experiencia de usuario (UX) al proporcionar feedback visual. No todos los formatos de animación son iguales; su elección depende del equilibrio entre calidad y peso del archivo.

GIF (Graphics Interchange Format): Es el formato más antiguo. Soporta transparencias binarias y bucles sencillos. Limitado a 256 tonos; ideal para memes o iconos simples.

CSS Animations: No es un archivo de imagen, sino código. Permite animar elementos HTML. Ligero y eficiente; el navegador lo procesa directamente.

SVG (Scalable Vector Graphics): Vectores basados en matemáticas. Ideales para logotipos e ilustraciones animadas sin pérdida de resolución.

Lottie (JSON): Formato moderno; exporta animaciones complejas de herramientas profesionales a archivos de código muy ligeros. Estándar actual.

WebP / APNG: Versiones modernas con mejores compresiones que el GIF tradicional, soportando millones de colores y transparencias parciales.

Herramientas de animación web

  • 1. Editores de código (VS Code): Para animaciones con CSS o librerías JS (GSAP). Ofrece control absoluto pero requiere programación.
  • 2. Adobe Animate: Crea animaciones vectoriales complejas exportables a HTML5 y Canvas.
  • 3. LottieFiles / After Effects: Diseña alta fidelidad y transforma en código mediante el plugin Bodymovin.
  • 4. Herramientas online (SVGator, Canva): Creación visual sin código, ideales para prototipos rápidos.

Nota técnica: En la animación web es crucial vigilar la tasa de refresco. Una animación fluida debe ejecutarse idealmente a 60 FPS (cuadros por segundo) para que el ojo humano no perciba saltos, lo cual exige un código optimizado que no sobrecargue la CPU del dispositivo.

Páginas web con Google Sites: entorno de desarrollo y paso a paso

Google Sites es una herramienta de creación de sitios web estructurada bajo el concepto WYSIWYG (What You See Is What You Get). A diferencia del desarrollo basado en código, permite construir sitios de manera visual y colaborativa.

Entorno de desarrollo

El entorno se divide principalmente en tres áreas:

  • Lienzo central: Donde se arrastran y organizan los elementos.
  • Panel lateral derecho: Herramientas de Insertar (textos, imágenes, Drive), Páginas y Temas.
  • Barra superior: Opciones de deshacer, vista previa, colaboración y publicación.

Paso a paso para la creación:

  1. 1. Planificación: Definir el mapa del sitio antes de abrir la herramienta.
  2. 2. Selección de Tema: Elección de línea estética para mantener la coherencia.
  3. 3. Maquetación por Bloques: Uso de bloques preconfigurados para asegurar diseño responsive.
  4. 4. Integración de Elementos: Inserción directa de formularios, mapas, vídeos o documentos.
  5. 5. Configuración de Navegación: Organización del menú superior o lateral.
  6. 6. Publicación: Asignación de URL y permisos de visibilidad.

Sabías que...

Aunque Google Sites no permite editar el código HTML profundo, tiene un botón llamado "Insertar código" que permite añadir pequeños scripts o elementos externos, permitiendo una personalización técnica avanzada dentro de un entorno simplificado.
