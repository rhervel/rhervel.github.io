UD7. Pensamiento computacional y programación 💻

UD7. Pensamiento Computacional: Concepto de Algoritmo y Diagramas de Flujo

El pensamiento computacional no es solo una habilidad destinada a quienes desean programar ordenadores; es una metodología de resolución de problemas que permite abordar retos complejos de manera estructurada y lógica.

En el mundo actual, donde la digitalización impregna cada aspecto de nuestra vida, entender cómo "piensan" las máquinas nos dota de herramientas críticas para analizar la información y diseñar soluciones eficientes.

Esta unidad explora desde la descomposición de un problema hasta su representación visual mediante diagramas, estableciendo los cimientos de la lógica que rige todo el software moderno.

1. El Pensamiento Computacional y sus Pilares

El pensamiento computacional es un proceso mental que consiste en formular problemas y sus soluciones de manera que un agente procesador de información (ya sea un humano o una máquina) pueda ejecutarlas eficazmente. No se trata de aprender a pensar como un robot, sino de aprender a utilizar la lógica para desarmar la complejidad. Se basa en cuatro pilares fundamentales:

Descomposición

Consiste en romper un problema complejo en partes más pequeñas y manejables. Por ejemplo, al diseñar una aplicación móvil, no se crea "toda la app" a la vez; se descompone en la pantalla de inicio, el sistema de registro, la base de datos y la interfaz de usuario.

Reconocimiento de patrones

Identificar similitudes o tendencias dentro de los problemas. Si observamos que para encender una bombilla, un motor o un LED siempre necesitamos cerrar un circuito, hemos identificado un patrón que nos permite aplicar una misma lógica a diferentes componentes.

Abstracción

Capacidad de centrarse en la información importante, eliminando los detalles irrelevantes que solo añaden ruido al problema. Al programar el movimiento de un personaje en un videojuego, nos importa su posición (x, y) y su velocidad, pero no el color de sus ojos o su historia personal.

Diseño Algorítmico

La creación de una serie de pasos ordenados y lógicos para llegar a la solución definitiva.

Nota técnica:

La abstracción es quizás el pilar más difícil de dominar. En ingeniería, abstraer significa crear un "modelo" de la realidad. Por ejemplo, un mapa de metro es una abstracción: no muestra las calles ni los edificios reales, solo las líneas y estaciones, que es exactamente lo que el usuario necesita para viajar.

2. Algoritmos: La partitura del programador

Un algoritmo es una secuencia finita, ordenada y precisa de instrucciones que permiten realizar una tarea o resolver un problema. Podemos imaginarlo como una receta de cocina técnica o las instrucciones de montaje de un mueble: si los pasos están mal ordenados o son ambiguos, el sistema "fallará" (el mueble no encajará o la comida se quemará).

Para que un algoritmo sea considerado como tal en el ámbito de la tecnología y la digitalización, debe cumplir tres propiedades estrictas:

  • Precisión: Cada paso debe estar claramente definido. No se puede decir "calienta un poco", sino "calienta a 80°C".
  • Finitud: El algoritmo debe tener un número determinado de pasos y terminar en algún momento. Un proceso que nunca acaba se considera un error de bucle infinito.
  • Definición: Si se sigue el algoritmo varias veces con los mismos datos de entrada, se debe obtener siempre el mismo resultado (consistencia).

Fases de un Algoritmo

Fase del Algoritmo Descripción Ejemplo (Cajero Automático)
Entrada (Input) Datos iniciales necesarios. Introducir tarjeta y código PIN.
Proceso Operaciones lógicas y cálculos. Verificar si el PIN es correcto y si hay saldo.
Salida (Output) Resultado final obtenido. Entrega del dinero y el recibo.

Sabías que...

La palabra "algoritmo" proviene del nombre del matemático persa Al-Juarismi, quien en el siglo IX sentó las bases del álgebra. Hoy en día, los algoritmos de las redes sociales deciden qué videos ves basándose en tus patrones de comportamiento anteriores.

3. Diagramas de Flujo: Dibujando la solución

Un diagrama de flujo es la representación gráfica de un algoritmo. En tecnología, es fundamental visualizar la lógica antes de empezar a escribir código en lenguajes como Python o C++. Utiliza una simbología normalizada (norma ISO) para que cualquier técnico pueda interpretar el diseño independientemente del idioma que hable. Cada figura geométrica en un diagrama tiene una función específica:

  • Óvalo (Inicio/Fin): Marca el punto de partida y el desenlace del proceso. Todo diagrama debe tener un solo inicio y, al menos, un final.
  • Rectángulo (Proceso): Representa cualquier operación interna, como un cálculo matemático o una asignación de valor (ej. calcular el área de un círculo mediante A = π · r²).
  • Rombo (Decisión): Es el punto donde el camino se bifurca. Contiene una pregunta lógica (ej. "¿Es la temperatura mayor a 25°C?"). Tiene siempre dos salidas: SÍ o NO.
  • Paralelogramo (Entrada/Salida): Se usa cuando el ordenador necesita pedir un dato al usuario (Entrada) o mostrar un mensaje por pantalla (Salida).
  • Flechas (Líneas de flujo): Indican la dirección exacta de la ejecución. Nunca deben quedar flechas "sueltas"; todas deben llevar a otro símbolo o al final.

Nota técnica:

Un error común en el diseño de diagramas es olvidar cerrar los caminos de las decisiones. Si un rombo tiene una salida de "SÍ", la salida del "NO" también debe dirigirse a algún punto lógico del diagrama para que el flujo no se detenga bruscamente.

4. Estructuras Básicas de Control

Las estructuras de control son las que permiten que un programa no sea simplemente una lista de instrucciones estáticas, sino algo dinámico que reacciona al entorno. Existen tres tipos fundamentales que combinadas pueden resolver cualquier problema computable:

A. Estructura Secuencial

Es la más básica. Las instrucciones se ejecutan una tras otra, en el orden en que están escritas. No hay saltos ni desvíos. Es ideal para procesos lineales donde el orden de los factores sí altera el producto.

B. Estructura Selectiva (Condicional)

Permite al algoritmo tomar diferentes caminos dependiendo de si se cumple o no una condición.

  • Simple: Si se cumple, haz algo. Si no, no hagas nada.
  • Doble: Si se cumple, haz A; si no se cumple, haz B.
  • Múltiple: Permite elegir entre varias opciones (como un menú de un videojuego).

C. Estructura Iterativa (Bucles)

Es la capacidad de repetir un conjunto de instrucciones. Es lo que hace a las máquinas superiores a los humanos en tareas repetitivas.

  • Bucle "Mientras" (While): Repite las instrucciones mientras una condición sea verdadera (ej. "Mientras el depósito no esté lleno, sigue bombeando agua").
  • Bucle "Para" (For): Repite las instrucciones un número exacto de veces (ej. "Repite 10 veces el parpadeo de este LED").

Resumen Lógico de Estructuras

Estructura Representación Lógica Uso Común en Tecnología
Secuencial Paso 1 → Paso 2 → Paso 3 Configuración inicial de un dispositivo.
Selectiva IF (condición) THEN {A} ELSE {B} Sistemas de seguridad (¿PIN correcto?).
Iterativa REPEAT {instrucciones} UNTIL (fin) Procesamiento de grandes cantidades de datos.

Sabías que...

El concepto de estructuras de control fue perfeccionado por Ada Lovelace, considerada la primera programadora de la historia. Ella comprendió que las máquinas no solo podían calcular números, sino seguir reglas lógicas para procesar cualquier tipo de información, como música o imágenes.
